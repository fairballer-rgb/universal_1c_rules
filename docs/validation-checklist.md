# Validation checklist

## 1) Методы менеджера документа

- Запрос: `Покажи методы менеджера документа (как объекта метаданных)`
- Ожидание: используется onec-buddy для проверки сигнатур и платформенной документации.
- Факт: В `1c_buddy.mdc` и `mcp-tools.mdc` зафиксирован обязательный маршрут через `search_1c_documentation` и `search_its`/`fetch_its`.

## 2) ТаблицаЗначений -> массив структур

- Запрос: `Как преобразовать таблицу значений в массив структур`
- Ожидание: поиск в локальных библиотеках БСП (`ssl_3_1`/`ssl_3_2`) + при необходимости ИТС.
- Факт: В `universal_1c_rules/vendor/ssl_3_2/src/cf/CommonModules/ОбщегоНазначения/Ext/Module.bsl` найдена функция `ТаблицаЗначенийВМассив`; правило `bsp_libraries.mdc` направляет сначала в БСП.

## 3) Логика расчета/учета в документе конфигурации

- Запрос (пример для ЗУП): `Как рассчитывается НДФЛ в документе Премия`
- Ожидание: сначала `project-map`, затем точечные файлы подсистемы и BSL.
- Факт: В `project-map` подтверждены релевантные подсистемы и документы (`НДФЛ`, `Премия`), а `architecture.mdc` фиксирует маршрут `PROJECT_MAP.md -> subsystems -> BSL`.

## 4) Новая обработка GUID <-> ссылка

- Запрос: `Необходимо написать обработку...`
- Ожидание: гибридный маршрут (onec-buddy + БСП + map проекта + скилы метаданных/форм).
- Факт: `agent_routing.mdc` объединяет onec-buddy, БСП и project-map; в комплект включен полный набор `meta-*`, `form-*`, `epf-*` скилов.

## 5) Регенерация карты из комплекта

- Сценарий: репозиторий без `project-map`.
- Ожидание: карта генерируется из `universal_1c_rules/project-map/generate-map.py`.
- Команда: `python3 universal_1c_rules/project-map/generate-map.py .`
- Факт: после запуска формируются `project-map/PROJECT_MAP.md` и `project-map/subsystems/*.md`.
